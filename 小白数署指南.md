# 小白专用！Vercel 部署超详细教程

> 本教程专为编程小白准备，跟着每一步做，一定能成功！
>
> 如果遇到问题，请先看文末的"常见问题"部分

---

## 📋 开始前的准备

你需要有：
- ✅ 一个 **GitHub 账号**（如果没有，先去 github.com 注册一个）
- ✅ 电脑能上网
- ✅ 有命令行工具（Windows 的 CMD、PowerShell，或 Mac 的终端）

---

## 第一步：安装 Vercel CLI 工具

### Windows 用户：

1. **按键盘上的 `Win + R` 键**，输入 `cmd`，按回车

2. **打开黑色窗口后**，复制下面这行命令（鼠标选中，右键复制）：

```bash
pnpm add -g vercel
```

3. **粘贴到黑窗口**（右键点击黑窗口空白处选择粘贴，或按 Ctrl+V）

4. **按回车键执行**

5. **等待安装**（可能需要 1-2 分钟），看到类似这样就算成功：
```
+ vercel@37.4.0
added 1 package in 2s
```

### 如果提示找不到 pnpm 命令？

这说明你没安装 pnpm，先执行：

```bash
npm install -g pnpm
```

然后再执行上面的安装 vercel 命令。

---

## 第二步：登录 Vercel 账号

1. **还是在刚才的黑色窗口里**，输入：

```bash
vercel login
```

2. **按回车**

3. **选择登录方式**，会看到类似这样的提示：

```
? Log in to Vercel
> Vercel
  GitLab
  Bitbucket
  Email
```

4. **按方向键（↑ ↓）** 选择 `Vercel`（默认就是 Vercel，直接按回车也可以）

5. **按回车**

6. **浏览器会自动打开**，如果没有自动打开，复制屏幕显示的链接到浏览器打开

7. **在浏览器里**：
   - 如果有账号，直接登录
   - 如果没有账号，点击 "Sign Up" 注册（推荐用 GitHub 登录，因为 GitHub 账号已经有了）

8. **登录成功后**，浏览器会显示 "Success! Logged in as xxx"，然后浏览器可以关掉

9. **回到黑色窗口**，会看到类似：
```
Success! Logged in as your-email@example.com
```

---

## 第三步：进入项目文件夹

1. **确认你在项目文件夹里**

黑色窗口应该显示类似：
```
C:\Users\你的用户名\projects>
```

或者显示其他路径，只要能看到 `package.json` 文件就行。

2. **如果不在项目文件夹**，用 `cd` 命令进入：

```bash
cd 你的项目文件夹路径
```

例如（Windows）：
```bash
cd C:\Users\Administrator\Desktop\projects
```

3. **确认是否正确**，输入：
```bash
dir
```

应该能看到 `package.json`、`src`、`vercel.json` 这些文件。

---

## 第四步：部署到 Vercel

### 1. 开始部署命令

在黑色窗口输入：

```bash
vercel
```

按回车

### 2. 第一个问题：是否设置并部署？

会看到：
```
? Set up and deploy "C:\Users\...\projects"? [Y/n]
```

**直接按回车**（默认是 Y，表示 Yes）

### 3. 第二个问题：选择哪个范围？

会看到：
```
? Which scope do you want to deploy to?
  your-team
> your-username
```

**直接按回车**（选择你的个人账号）

如果显示团队账号，用方向键选择你的用户名，然后回车

### 4. 第三个问题：链接到现有项目吗？

会看到：
```
? Link to existing project? [y/N]
```

**按 `n` 键，然后回车**（我们创建新项目）

### 5. 第四个问题：项目叫什么名字？

会看到：
```
? What's your project's name?
```

输入一个英文名字，例如：
```
neimenggu-zsb
```

然后按回车

**注意**：
- 只能用英文、数字、横线
- 不能用中文
- 不能有空格

### 6. 第五个问题：代码在哪个目录？

会看到：
```
? In which directory is your code located?
```

**直接按回车**（默认是当前目录）

### 7. 第六个问题：要修改这些设置吗？

会看到一堆配置信息，最后问：
```
? Want to modify these settings?
```

**按 `n` 键，然后回车**（使用默认配置）

---

## 第五步：等待部署

现在 Vercel 会开始自动部署，你会看到类似这样的输出：

```
Vercel CLI 37.4.0
Preparing deployment...
...
```

### 部署过程（1-2 分钟）：

1. **检测项目** - 自动识别是 Next.js 项目
2. **安装依赖** - 自动安装所需的包（pnpm install）
3. **构建项目** - 自动编译项目（pnpm run build）
4. **部署到云端** - 上传到 Vercel 服务器

### 如果遇到错误：

看到红色文字或 ERROR，先别慌，看错误信息：

**常见错误 1：pnpm: command not found**

说明没安装 pnpm，执行：
```bash
npm install -g pnpm
```
然后重新运行 `vercel` 命令

**常见错误 2：Build failed**

构建失败，可能是：
- 代码有问题（但我们已经测试过了）
- 网络问题
- 暂时性错误

**解决方法**：重新执行 `vercel` 命令

## 💡 更新网站内容

以后如果需要修改网站内容（比如更新数据），只需要：

1. **修改代码或数据**

## ❓ 常见问题解答

### Q1: 黑色窗口输入命令没反应？

**A:** 可能是：
- 命令输错了，重新输入
- 没有按回车，输入完要按回车
- 窗口卡住了，关闭重新打开

### Q2: 提示 "command not found: vercel"

**A:** vercel 没安装成功，重新执行：
```bash
pnpm add -g vercel
```

如果还是不行，试试：
```bash
npm install -g vercel
```

### Q3: 登录 Vercel 时打不开浏览器？

**A:** 手动复制屏幕上的链接，用浏览器打开：
```
https://vercel.com/cli/login
```

### Q4: 部署时卡住了不动？

**A:**
- 按一次 `Ctrl + C` 停止
- 重新执行 `vercel` 命令
- 可能是网络问题，等一会儿再试

### Q5: 部署成功了，但网址打不开？

**A:**
- 等待 1-2 分钟（刚部署好可能需要几秒钟生效）
- 检查网址是否复制正确
- 尝试用其他浏览器打开
- 如果还是不行，重新部署一次

### Q6: 显示 "Error: listen EADDRINUSE"

**A:** 本地端口被占用，不影响部署，直接忽略这个错误。

### Q7: 可以修改网址吗？

**A:** 可以，在 Vercel 网页端修改：
1. 访问 vercel.com 登录
2. 进入你的项目
3. Settings → Domains
4. 可以添加自定义域名（需要有自己的域名）

### Q8: 要付费吗？

**A:** 不需要！Vercel 免费版足够使用：
- 每月 100GB 流量
- 无限项目
- 自动 HTTPS
- 全球 CDN

### Q9: 数据会丢失吗？

**A:** 不会，数据存在 Vercel 的服务器上，永久保存。

### Q10: 别人能看到我的代码吗？

**A:**
- 如果你把代码放在 GitHub 公开仓库，别人能看到
- 如果代码在你本地，部署到 Vercel 后，别人只能访问网页，看不到代码
- 数据库内容（招生计划等）别人只能查询，看不到原始数据
